Admin Control Panel — Proposed Features

Select all features you want in scope. Deselect anything out of scope.

1

Block / Unblock User

Prevent a specific user from triggering any jobs. Blocked users see a banner with a reason. Supports optional auto-expiry ("blocked until X").

2

System-Wide Lock

One toggle that blocks ALL users simultaneously. Useful for maintenance, deployments, or critical bugs. All job triggers return a system-locked error. Optional unlock time.

3

User Message / Banner

Show a persistent banner to one or more users in their dashboard (info / warning / error severity). User can dismiss it. Supports expiry time.

4

System Broadcast

Push a message to ALL users simultaneously. Shown as a top banner on every page. Ideal for "known issue", "maintenance window", "upgrade complete" announcements.

5

Kill Active Job

Forcibly mark a running job as failed from the admin panel. Useful when a job is stuck or runaway. Triggers watchdog-style update on the job file.

6

Read-Only Mode (per user)

User can browse dashboard and see verdicts/portfolio, but all job triggers are disabled with a "contact admin" message. Less severe than a full block.

7

Rate Limit Override / Zero Out

Instantly set a user's remaining quota to 0 for a job type (e.g. deep_dive), effectively blocking just that action without a full account block.

8

Force Session Invalidation

Invalidate a user's JWT by rotating their personal token secret. They're logged out immediately on next API call. Nuclear option for security incidents.
